# 碎股处理机制

## 碎股的产生来源

分红配股后，`_before_market`（`backtest.py:149`）处理送股：

```python
new_amount = int(entry["cash_stk"])  # 截尾取整，券商按整股派发
pos.amount += new_amount
```

`cash_stk` 是 `持仓量 × stk_div_per_share`（每股送股比例），比如持仓 300 股、每股送 0.3 股 → 送 90 股，加完后变 390 股，不是 100 的整数倍，碎股就产生了。

---

## 处理方式

### 1. 开盘前强制清仓

`_morning_start`（`backtest.py:162~184`）专门处理持仓总量不足一手的情况：

```python
# 散股检测：持仓数量不足一手（分红配股后可能产生）
min_lot = 200 if code.startswith("688") else 100
if 0 < pos.amount < min_lot:
    frac_codes.append(code)

# 散股强制清仓：用当日行情价，获得现金等价
for code in frac_codes:
    force_sell(code, ctx)
```

注意这里只处理**持仓总量不足一手**的情况（比如只剩 90 股）。如果持仓是 390 股，这里不会触发。

### 2. 卖出时的散股兜底

`rule_volume_num`（`rules.py:189~194`）在卖出时还有一个兜底：

```python
# 避免遗留零散股：卖出后剩余不足一手时，自动清仓
remaining = pos.amount - order.amount
min_lot = 200 if code.startswith("688") else 100
if 0 < remaining < min_lot:
    order.amount = pos.amount  # 直接改成全部卖出
```

比如持仓 390 股，用户想卖 300，剩余 90 < 100，系统自动把卖出量改成 390 全清。

---

## 汇总

| 场景 | 处理方式 |
|------|----------|
| 送股后总持仓 < 一手 | `_morning_start` 开盘前 `force_sell` 强制清仓 |
| 卖出后剩余 < 一手 | `rule_volume_num` 自动扩量到全清 |
| 送股后总持仓 >= 一手但非整百 | 不处理，允许持有（如 390 股） |

最后一种情况是设计上的取舍：送股后持仓 390 股是合法的，真实券商也允许持有，只是下次买入/卖出时会被 `rule_volume_num` 按整手取整。
