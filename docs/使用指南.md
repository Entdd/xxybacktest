# xxybacktest 使用指南

## 一、系统概述

xxybacktest 是一套**量化策略从研发到每日自动运行的完整闭环**。用户写一次策略代码，注册为模拟账户后，系统每天自动重跑回测、更新净值曲线、生成当日交易信号。打开 Web 界面就能看到"今天该买什么卖什么"，直接跟单操作。

### 核心功能

- **策略回测**：基于历史数据验证策略表现
- **模拟账户**：将策略注册为每日自动运行的模拟交易账户
- **定时任务**：灵活调度数据更新、策略重跑等任意脚本
- **Web 可视化**：策略净值、持仓、信号一目了然

### 架构简图

```
xxy-sim 启动
└── Flask (port 5000)
    ├── /              ← 账户列表
    ├── /account/<id>  ← 账户详情（净值曲线、持仓、信号）
    └── /tasks         ← 定时任务管理

APScheduler（后台）
    ├── builtin_run_simulation  ← 内置：每日模拟交易重跑
    ├── task_数据更新            ← 用户注册：数据更新脚本
    └── task_xxx                 ← 用户注册：任意脚本
```

## 二、快速开始

### 2.1 安装

```bash
# 在 conda 环境中安装
conda activate vnpy
cd D:/Desktop/github汇总/xxybacktest
pip install -e .
```

### 2.2 启动服务

```bash
# 方式一：使用快捷脚本
python run_simulation.py --data ./data --time 22:00

# 方式二：使用安装后的命令
xxy-sim --data ./data --time 22:00
```

参数说明：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--data` | 数据目录路径 | `./data` |
| `--time` | 每日模拟交易触发时间（HH:MM） | `22:00` |

启动成功后，打开浏览器访问 http://localhost:5000

## 三、提交策略（注册模拟账户）

### 3.1 编写策略代码

策略由两个函数组成：

```python
from xxybacktest.simulation import submit

def initialize(context):
    """初始化函数，策略注册时执行一次"""
    # 加载历史数据、设置参数等
    pass

def handle_data(context):
    """每日调仓函数，每天收盘后执行一次"""
    # 选股、调仓等逻辑
    # 示例：全仓买入
    context.order_target_percent("000001.SZ", 1.0)
```

### 3.2 注册为模拟账户

```python
from xxybacktest.simulation import submit

account_id = submit(
    name="我的策略",              # 账户名称
    initialize=initialize,      # 初始化函数
    handle_data=handle_data,    # 调仓函数
    capital=100000,             # 初始资金（10万）
    start_date="2024-01-01",    # 策略开始日期
    data_path="./data",         # 数据目录
    asset_type="stock",         # 资产类型
    benchmark="000001.SH",      # 基准指数
    run_now=True,               # 注册后立即跑一次回测
)

print(f"账户已注册: {account_id}")
```

### 3.3 立即运行单账户回测

注册时设置 `run_now=True` 会立即执行一次回测。如果注册时没有立即运行，之后可以手动触发：

```python
from xxybacktest.simulation import run_single

result = run_single(account_id, data_path="./data")
print(result)
```

### 3.4 账户管理

```python
from xxybacktest.simulation import (
    list_accounts, get_account,
    pause, resume, delete
)

# 列出所有账户
accounts = list_accounts(data_path="./data")

# 查看单个账户详情
info = get_account(account_id, data_path="./data")

# 暂停账户（暂停后不再参与每日重跑）
pause(account_id, data_path="./data")

# 恢复账户
resume(account_id, data_path="./data")

# 删除账户及其所有历史数据
delete(account_id, data_path="./data")
```

## 四、Web 界面

启动服务后访问 http://localhost:5000

### 4.1 账户列表页（/）

- 展示所有模拟账户的**累计收益率、当日收益、最大回撤**
- 点击账户卡片进入详情页
- 支持**策略组合**：多选账户后按权重合成组合净值曲线

### 4.2 账户详情页（/account/<id>）

- **净值走势**：策略净值 + 基准指数 + 超额净值三条曲线
- **计划交易**：最新调仓信号（今天该买什么/卖什么）
- **当前持仓**：代码、名称、数量、成本、市值、占比、累计收益
- **历史成交**：带分页的完整交易记录
- **操作按钮**：暂停 / 恢复 / 删除账户

### 4.3 任务管理页（/tasks）

- 查看所有定时任务的**名称、cron 表达式、下次执行时间、上次状态**
- **手动触发**：立即执行一次任务
- **查看日志**：弹出窗口实时展示任务执行日志（3秒轮询刷新）
- **删除任务**：用户注册的任务可删除，内置任务不可删除

## 五、定时任务

系统使用 APScheduler 管理定时任务，支持**热注册、热删除**，无需重启服务。

### 5.1 内置任务：每日模拟交易

启动 `xxy-sim` 时自动注册，通过 `--time` 参数控制触发时间：

```bash
xxy-sim --data ./data --time 22:00   # 每晚 22:00 自动重跑所有运行中账户
xxy-sim --data ./data --time 15:30   # 收盘后 15:30 执行
```

触发后会并行执行所有**运行中**账户的回测，每个账户结果独立存储。

### 5.2 注册自定义定时任务

```python
from xxybacktest.simulation import schedule_task

# 方式一：每天固定时间
tid = schedule_task(
    name="数据更新",
    script="./data_renew.py",   # 脚本的绝对或相对路径
    time="17:30",                # 每天 17:30 执行
    data_path="./data",
)

# 方式二：cron 表达式
tid = schedule_task(
    name="季报数据",
    script="./update_quarterly.py",
    cron="0 10 1 1,4,7,10 *",    # 每季度第一天 10:00
    data_path="./data",
)
```

**参数说明**：

| 参数 | 说明 |
|------|------|
| `name` | 任务名称（展示用） |
| `script` | Python 脚本路径 |
| `time` | 每天固定时间，格式 `HH:MM`，与 `cron` 二选一 |
| `cron` | 标准 5 段 cron 表达式（分 时 日 月 周），与 `time` 二选一 |
| `data_path` | 数据目录路径 |

**注意事项**：
- `time` 和 `cron` 只能指定一个
- `script` 会被自动转为**绝对路径**存储，避免重启后路径失效
- 相同 `name + script + cron` 的任务不会重复注册

### 5.3 删除任务

```python
from xxybacktest.simulation.task_store import remove_task

remove_task(tid, data_path="./data")
```

或者在 Web 界面的 **/tasks** 页面点击删除按钮。

### 5.4 cron 表达式参考

| 需求 | cron 表达式 |
|------|------------|
| 每天 22:00 | `0 22 * * *` |
| 每天 17:30 | `30 17 * * *` |
| 工作日 9:00 | `0 9 * * 1-5` |
| 每周一 10:00 | `0 10 * * 1` |
| 每月 1 日 9:00 | `0 9 1 * *` |
| 每季度首日 10:00 | `0 10 1 1,4,7,10 *` |

## 六、编程接口参考

### 6.1 账户管理

```python
from xxybacktest.simulation import (
    submit, pause, resume, delete,
    list_accounts, get_account,
    run_all, run_single,
    get_account_nav, get_account_positions, get_account_orders,
    schedule_task,
)
```

| 函数 | 说明 |
|------|------|
| `submit(...)` | 注册策略为模拟账户 |
| `pause(account_id)` | 暂停账户 |
| `resume(account_id)` | 恢复账户 |
| `delete(account_id)` | 删除账户及历史数据 |
| `list_accounts(status)` | 列出账户，`status='running'` 只列运行中 |
| `get_account(account_id)` | 获取单个账户详情 |
| `run_all(end_date)` | 重跑所有运行中账户 |
| `run_single(account_id, end_date)` | 重跑单个账户 |
| `get_account_nav(account_id)` | 获取账户净值 DataFrame |
| `get_account_positions(account_id, date)` | 获取持仓快照 |
| `get_account_orders(account_id, limit)` | 获取成交记录 |
| `schedule_task(...)` | 注册定时脚本任务 |

### 6.2 HTTP API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/tasks/api/list` | GET | 获取所有定时任务状态 |
| `/tasks/api/trigger/<task_id>` | POST | 手动触发一次任务 |
| `/tasks/api/delete/<task_id>` | POST | 删除用户任务 |
| `/tasks/api/log/<task_id>` | GET | 获取任务最近一次日志 |

## 七、从旧版迁移

如果你之前使用 `xxy-sim --data-renew` 启动（旧版 Plombery 内置了数据更新任务），请按以下方式迁移：

**旧版启动方式（已废弃）：**
```bash
# 不再支持
xxy-sim --data-renew ./data_renew.py --time 22:00
```

**新版方式：**
```bash
# 1. 先启动服务（只负责模拟交易调度）
xxy-sim --data ./data --time 22:00

# 2. 在另一个 Python 会话中注册数据更新任务
from xxybacktest.simulation import schedule_task
schedule_task(name="数据更新", script="./data_renew.py", time="17:30", data_path="./data")
```

**数据存储格式变更：** 模拟结果从"所有账户共享表"改为"每账户独立 Parquet 文件"，存储路径为：

```
data/simulation_results/accounts/
  sim_001/daily_values.parquet
  sim_001/positions.parquet
  sim_001/orders.parquet
```

旧版数据（共享表）需要迁移，最简单的方式是**让所有账户重跑一次**，新文件会自动生成。

## 八、常见问题

**Q：Flask 的 debug 模式可以开吗？**

A：不可以。APScheduler 的 `BackgroundScheduler` 在 `debug=True` 时会被 reloader 启动两次，导致调度器重复运行。`main.py` 已固定 `debug=False`。

**Q：手动触发任务时提示"任务正在执行中"怎么办？**

A：同一任务设置了 `max_instances=1`，如果上次还没跑完，手动触发会被拒绝。请等待上次执行完成，或查看日志确认进度。

**Q：Windows 下并行执行有问题吗？**

A：Windows 使用 spawn 模式启动子进程，功能正常但启动开销比 Linux fork 大。如果账户数很少（<4），并行收益不明显。

**Q：日志在哪里看？**

A：任务日志存储在 `data/simulation_results/task_logs/<task_id>.log`，可以通过 Web 界面的"日志"按钮实时查看。
